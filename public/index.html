Group contract
GitHub: Regler som tildeles vores arbejde med GitHub:
  • Der arbejdes aldrig direkte til master. a. Der skal pushes på en ’XXX’-branch, som skal sikre, at der KUN ligger kode på master som virker. b. Der skal laves relevante branches for det arbejde, som man påbegynder. c. Branches skal slettes igen efter grøn test, push til master og endt arbejde med problemstilling. d. Der skal skrives i Discord-chatten inden man pusher ny kode op.
Mødepunkter:
  • Mødetid er klokken 10.00 om morgenen. Dette tilpasses efter behov og kommunikeres på Discord.
  • Alle gruppemedlemmer møder til gruppemøde om morgenen klokken 10.00. Her gennemgås gårsdagens opgaver, eventuelle problemer og løsninger.
  • Planer og udførelsen af disse for den pågældende dag drøftes i plenum.
  • Til gruppemødet er det vigtigt, at det sidste punkt vi gennemgår, er om samarbejdet i gruppen forløber som forventet, eller om vi skal omdirigere ressourcer.
Kommunikation:
  • Al kommunikation foregår på gruppens Discord-chat eller Zoom, hvis dette er nødvendigt.

Request/Response formats

  1. Person (omit id from POST and PUT methods)

    {
    ‘id’: number,
    ‘email’: String,
    'phones': [phone,phone,...](2),
    ‘firstname’: String,
    ‘lastname’: String,
    ‘address’: address(3),
    ‘hobbies’: [hobby, hobby, …](4),
    ‘zip’: number
    }

  2. Phone

    {
    'number': number,
    'description': String
    }

  3. Address

    {
    'street': String,
    'info': String
    }

  4. Hobby (omit everything but id for PUT and POST methods)

    {
    'id' number,
    'name': String,
    'link': string
    }
